# Documentation Structure

This folder contains comprehensive documentation for the trading application organized in a modular structure.

## Folder Organization

### 📋 standards/
Coding standards, best practices, and quality assurance documentation:
- `LINE_BY_LINE_STANDARDS.md` - Standards for every single line of code
- `IMPLEMENTATION_MICRO_STANDARDS.md` - Micro-level implementation patterns
- `CODE_REVIEW_CRITERIA.md` - Comprehensive review checklist
- `CODE_TEMPLATES.md` - Copy-paste templates for common patterns
- `CODING_CHECKLIST.md` - Quick verification before submission
- `CLAUDE_REVIEW_WORKFLOW.md` - 5-phase code review process

### 🏗️ architecture/
System architecture and technical design documents:
- `TRADING_SYSTEM_ARCHITECTURE.md` - Overall system architecture overview
- `REALTIME_DATA_FLOW_ANALYSIS.md` - Real-time data flow and WebSocket architecture
- `PERSISTENT_WEBSOCKET_SOLUTION.md` - WebSocket connection management strategy

### 📖 guides/
Implementation guides and operational documentation:
- `DEPLOYMENT_GUIDE.md` - Deployment and infrastructure setup
- `AUTO_STOCK_TRADING_README.md` - Auto trading system documentation
- `BROKER_FUNDS_IMPLEMENTATION.md` - Broker integration and funds management
- `MARGIN_INTEGRATION_SUMMARY.md` - Margin trading integration details
- `SECURITY.md` - Security practices and guidelines

### 🗓️ planning/
Project planning, roadmaps, and TODO lists:
- `FEATURE_ROADMAP_TODO.md` - Feature development roadmap
- `AUTO_TRADING_IMPLEMENTATION_TODO.md` - Auto trading implementation tasks

## Quick Reference

For developers working on this project:

1. **Before writing code**: Review all files in `standards/` folder
2. **Architecture questions**: Check `architecture/` folder
3. **Implementation help**: Refer to `guides/` folder
4. **Project planning**: See `planning/` folder

## Main Configuration

The main project configuration and overview remains in the root `CLAUDE.md` file, which references these organized documentation files.
